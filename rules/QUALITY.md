# QUALITY ルール — 品質バー（評価6観点）と相対評価・自己改善

作業の「良し悪し」をどう測り、どう改善し続けるかの決まり。各ノードは作業中にこの観点を意識し、
**`retro` Skill** が完了後にこのルーブリックで採点して Skill/rule を改善する。
全体像は [WORKFLOW.md] §6。採点の生データは status.json の `metrics`（[WORKFLOW.md] §0）。

## 評価6観点（ユーザー指定の品質バー）

| # | 観点 | 何を見るか（悪い兆候） | 主なシグナル源 |
|---|---|---|---|
| 1 | **手戻り（rework）** | 修正・指摘で戻りが発生していないか | verify FAIL 回数 / review 重大度・高 件数 / 「レビュー対応」commit / マージ後に同 issue を直す commit / `metrics.loopbacks` |
| 2 | **速度（speed）** | 完了まで**無駄に**遅くなっていないか | `metrics.duration_sec`（ノード/全体）をタスク規模に照らす |
| 3 | **網羅性・検証充足（coverage）** | **逆に薄すぎないか**＝検証・網羅が足りているか | coverage% / 追加テスト有無 / 状態網羅（空/ロード/エラー）/ ゲート充足 / 受入基準カバー |
| 4 | **トークン妥当性（tokens）** | トークン消費が**不当に高くないか** | `metrics.tokens.total` をタスク規模に照らす（自己申告/transcript・best-effort） |
| 5 | **成果物クオリティ（artifact_quality）** | 成果物の質が高いか | review findings の重大度分布 / マージ後不具合 / 簡潔さ・規約適合 |
| 6 | **抜け漏れ（completeness）** | 作業の抜け漏れがないか | 未チェックの必須チェックリスト / 受入基準の未達 / docs 同期漏れ / commit 規約 |

**②速さ と ③網羅性は相反**する（速い＝検証が薄い、網羅＝冗長になりうる）。両者の**バランス**を必ず併せて見る
（速いだけ・厚いだけを高評価しない）。

## 採点方式 — 上限なしの加点・**前回比の相対評価**

絶対100点満点は「常に100点に張り付いて無意味」になるため**採らない**。代わりに:

- **上限を設けない加点ポイント制**で、観点ごとに「**前回（直近の基準値／移動平均）と比べてどうだったか**」を
  加点/減点する（改善=＋、悪化=−）。累積ポイントは積み上がり続け飽和しない＝「改善し続けているか」を見る。
- 基準値と累積は **`docs/retro/scoreboard.json`**（`paths.retro_dir`）に保持。各 retro は観点別に
  `前回比 +X / −Y` と累積ポイントを `docs/retro/<date>-<scope>.md`（雛形 [.claude/templates/retro-report.md]）に出す。
- 規模補正: tokens/duration は「タスク規模」で正規化してから比較する（小タスクと大タスクを同列にしない）。

## 改善の検証 — ブラインド A/B（修正バイアスの排除）

「自分が直したから良くなったはず」というバイアスを避けるため、Skill/rule の改善は**ブラインド対戦で検証**してから採用する:

1. 改善案を**修正後 Skill** として用意（修正前 Skill は git で復元可能）。
2. 代表タスクを **修正前 / 修正後 の双方**で実施し、2つの成果物を得る。
3. 2成果物を**匿名化（成果物A / 成果物B）・順序ランダム化**し、**どちらが修正後かを伏せて**
   別の評価 agent（`eval-judge`）にブラインドで渡す（A/B↔前後の対応表は判定後まで開示しない）。
4. `eval-judge` は本6観点で**ペア比較**し、勝者と差分を返す。
5. **修正後が勝ったときだけ採用**（負け/引き分けは破棄 or 再設計）。採用後 `run-evals.sh` を再実行し、
   対応する `eval/spec.json` の `must_contain` に新不変項を追加して**再発を機械検出で固定**する。

ランダム化・封印は retro のオーケストレーション（メインループ）が担い、対応表を `eval-judge` に渡さない
（評価の独立性を担保）。詳細手順は `.claude/skills/retro/SKILL.md`。
